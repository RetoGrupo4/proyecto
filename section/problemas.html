PROBLEMAS ENCONTRADOS PARA HACER LA WIKI

Instalación del Proxmox →

No encontrábamos un programa booteable para hacer la instalación a través del USB, utilizamos el RUFUS y el UNETBOOTIN y no nos funcionaba; no era compatible.

Probamos a través de comandos que buscamos en Internet pero tampoco llego a funcionar. Leímos que en windows se hacia a través de ETCHER y miramos el programa y vimos que también estaba para Ubuntu, lo descargamos y hicimos el USB de instalación.

Problemas con el Proxmox→

No sabíamos como crear un contenedor porque tampoco sabíamos como funcionaban lo de las plantillas. Al principio descargamos una ISO y no valía para crear un contenedor, se necesitaba una plantilla. Una vez descargada la plantilla ya se pudo crear el contenedor.

Problemas de compatibilidad con Chrome

Al principio vimos que al abrir en local (file://protocolo) con Firefox se veia perfectamente; sin embargo en Chrome no aparecía ni el section ni el aside. Pensabamos que era problema de estilos en el navegador, probamos reseteando, mirando cualquier codigo, pero no solucionaba el problema. Abrimos en local (file://protocolo) en el chrome y le dimos a F12 para ver si había algun error que apareciese consola. Vimos que aparecia un error de javascritpt

El error indica, que las solicitudes solo están soportadas por protocolos http , data, chrome, chrome-extension, https.

En Firefox no nos sale ese problema porque si permite utilizar solicitudes AJAX usando el file://protocolo , el Chrome sin embargo, bloquea esto por defecto.

Es posible habilitar una configuración en Google Chrome que permita solicitudes al sistema de archivos local desde el navegador ( allow-file-access-from-files) pero no se recomienda usar.

Para que esta solicitud AJAX funcione en Chrome, debe realizar la solicitud a un servidor web ( XAMPP, ISS, NODEJS).
